Detailed Description

This interface allows control of an LED interface of up to 16 LEDs.

Summary

The driver supports High Level or direct implementation for controlling up to 16 LEDs. This High Layer requires an associated low layer implementation specific to the platform. The HLD exposes a global for the LED driver, this must only be accessed directly if used in an OS Free implementation.

Before accessing the driver, call the open function to configure the selected hardware interface, this must be implemented in the Lower Driver layer. When open the control function can be used to control the status of the LEDS.

When finished with the driver close it using the close function.

This driver does not allocate any heap memory.

This driver does use locking primitives to manage access to the user LEDs.

led_get_version - Get driver version

Generic example of driver usage st_def_stdio_device_t

Known Limitations
Only LED 0 is supported by this driver on the RZA1LU Stream-IT board for the User LED.

Known Implementations
This driver is used in the RZA1LU Software Package. RENESAS_APPLICATION_SOFTWARE_PACKAGE
